🧩 1. Si no se genera el archivo .raw

Síntoma: El script te dice

“No se generó el archivo config_X.raw”

🔍 Qué revisar

LTspice no está donde el script lo busca.
→ Verificá la ruta del ejecutable:

LTSPICE_EXE = r"C:\Program Files\LTC\LTspiceXVII\XVIIx64.exe"


Si tenés LTspice XVIII, la ruta puede ser algo como
C:\Program Files\Analog Devices\LTspice\LTspice.exe

La simulación no termina correctamente.
→ Probá abrir manualmente el .asc generado (resultados/config_X.asc)
y corrélo en LTspice.
Si hay errores de conexión o modelos faltantes, te los muestra.

Comando de batch falló.
Podés agregar temporalmente print(result.returncode) o check=False en el subprocess.run para ver si LTspice está devolviendo un error.

⚙️ 2. Si el .raw se genera pero el .csv sale vacío o incompleto

Síntoma: El CSV tiene solo la columna t, o faltan corrientes/tensiones.

🔍 Qué revisar

Nombres de trazas (curvas) distintas.
→ Abrí el .raw en LTspice manualmente y mirá cómo se llama la corriente:
a veces LTspice la nombra I(VCELL11) (en mayúsculas) o I(Vcell_11).
Si cambia el formato, solo hay que ajustar esta línea en el script:

nombre = f"I(Vcell{i}{j})"


por ejemplo a:

nombre = f"I(VCELL{i}{j})"


No hay corriente medida (fuente no conectada o apagada).
Si un switch está en VcOFF, puede que la fuente de esa celda no conduzca y la corriente sea 0 o ni aparezca en el .raw.
En ese caso, PyLTSpice simplemente no la encuentra.

Nombres de nodos de salida.
Verificá que los nodos en tu netlist realmente se llamen out_pos y out_neg.
Si se llaman distinto, ajustá:

"V(out_pos)" / "V(out_neg)"


a los nombres correctos.

⚡ 3. Si tarda mucho o LTspice se congela

LTspice a veces se queda en análisis transitorio por pasos muy chicos.
Podés acortar la simulación en tu archivo base .asc (por ejemplo tran 0 2m 0 1u).

Evitá guardar muchas combinaciones grandes al principio (empezá con 1 o 2 filas).

🧠 4. Si querés depurar una sola combinación

Podés forzar el script a correr solo una configuración específica editando:

combinaciones = [tu_mapping_unico]


o directamente:

combinaciones = [generar_combinaciones()[2]]


para correr solo la tercera configuración de la lista.

📊 5. Si el análisis (plot_resultados.py) no muestra curvas

Revisá que los .csv realmente tengan columnas con datos (abrilos con Excel o pandas).

Asegurate de ejecutar el script desde la misma carpeta donde está la carpeta resultados/.

Si te dice “faltan columnas de salida”, es porque no se registraron V_out_diff o I_RL.
→ Revisá que tu .asc tenga esos nodos y resistencia RL bien definidos.
